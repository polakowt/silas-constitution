**Why:** Tom blew the Lucid eval. No prop firm = no eval rules needed. Simplifying the system removes constraints that were only relevant to a context that no longer exists. Dry-run preserves strategy development without real capital at risk.
**Constitutional basis:** Truth — acknowledge the situation accurately and adapt. Stewardship — capital protection through dry-run while strategy matures.

### 2. OVN bot scraper fixed — levels extraction restored
**Decided:** Fixed Discord scraper to auto-navigate to correct channel, use full-res CDN URL (822×1398 vs 165×280 preview), and fixed HP/MHP sed regex collision. Added 3× retry with 30s backoff and Telegram alert on final failure.
**Why:** Four+ consecutive nights of level extraction failure traced to three root causes: wrong channel, low-res preview image, and a regex bug where `HP=` matched `MHP=`. All fixable. Fixed them.
**Constitutional basis:** Stewardship — a broken tool that can be fixed should be fixed. Integrity — don't run a system on bad inputs.

### 3. Sell fail protection added — auto-halt after 2 consecutive failures
**Decided:** Added consecutive sell fail counter to `ovn_runner.py`. Bot halts and sends Telegram alert after 2 consecutive sell button failures. Added MotiveWave auto-fix: on first sell fail, scans all MW windows for order buttons and refocuses.
**Why:** Saturday night the bot fired 20+ SELL signals, all failed silently — no capital lost because dry-run, but the failure mode is dangerous in live trading. The fix creates a circuit breaker.
**Constitutional basis:** Stewardship — prevent runaway broken execution. Integrity — a bot that fires signals it can't execute is lying to itself.

### 4. HP breakout entry logic added

**Constitutional basis:** Stewardship — learn from what was missed and build the capability. Truth — if the strategy says breakouts are valid entries, the code should reflect that.
